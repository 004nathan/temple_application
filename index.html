🔒 Firestore Security Rules
TEST MODE (Current) Use while building — no restrictions 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

⚠️ Anyone on the internet can read/write your data. Fine for internal tools, not for public apps.

PRODUCTION MODE (Recommended) Staff only — requires Firebase Auth sign-in 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /functions/{id} {
      allow read, create, update, delete: if request.auth != null;
    }
    match /dailyRecords/{id} {
      allow read, create, update, delete: if request.auth != null;
    }
  }
}

✅ Only signed-in users can read or write. Enable Authentication → Email/Password in Firebase Console and add staff users.

Updates show “Missing or insufficient permissions”? In the Firebase Console → Firestore → Rules, make sure writes use request.auth != null (signed-in allowed). An older template used request.auth == null, which blocks signed-in staff from saving. Copy the rules above exactly and click Publish.

How to apply rules in Firebase Console

  1. Go to console.firebase.google.com
  2. Select project temple-manager-93cc6
  3. Click Firestore Database in the left sidebar
  4. Click the Rules tab at the top
  5. Paste the rules above → Click Publish
🌐 Hosting Options
🔥 Firebase Hosting Recommended · Free
URL: temple-manager-93cc6.web.app
# Install Firebase CLI (once)
npm install -g firebase-tools
# Login to Firebase
firebase login
# Create a folder, put your HTML file in it, then:
firebase init hosting
# Deploy live
firebase deploy

When asked "What do you want to use as your public directory?" → type / (current folder). Single page app → No.

⚡ Netlify Drag & Drop · Zero setup
  1. Go to app.netlify.com → Log in / Sign up (free)
  2. Click "Add new site" → "Deploy manually"
  3. Drag and drop your temple-management.html file
  4. You instantly get a live URL like random-name.netlify.app
  5. Optionally set a custom domain in Site Settings
🐙 GitHub Pages Free · Permanent
  1. Create a new repo on github.com
  2. Upload temple-management.html — rename it to index.html
  3. Go to repo Settings → Pages
  4. Under "Branch" select main → Save
  5. Live at https://username.github.io/repo-name

ScenarioImpactFix
Switch to production mode without updating rules 🔴 App breaks All reads & writes blocked. Paste the production rules above first.

Test mode expires (30 days) 🔴 App breaks Paste either set of rules before expiry to avoid downtime.
